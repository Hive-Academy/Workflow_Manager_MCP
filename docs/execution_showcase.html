Context Preservation Principles

Immediate Linking

Plan creation and subtask generation happen in the same step while context is fresh and task intent is clear.

Iterative Validation

Subtasks are immediately validated against task goals, with refinement loops to ensure perfect alignment.

Context Boundaries

Clear exclusion rules prevent documentation and meta-analysis tasks from being created when not needed.

Embedded Guidance Architecture

Database-Driven Rules

Workflow rules stored in database, not static files, enabling dynamic updates and context-aware guidance.

Embedded Intelligence

Every MCP response includes contextual guidance, eliminating need for external rule references.

Shared Services

Centralized guidance generation eliminates 90% code duplication across MCP services.
